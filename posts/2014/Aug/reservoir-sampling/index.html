Reservoir Sampling

Mon 04 August 2014

Randomly choose $k$ samples from a list of $n$ items where $n$ is either unknown or very large. Assume that this data stream might not fit into main memory. Furthermore, I should take $O(n)$ time and $O(k)$ space.

$O(k)$ space is what makes the problem interesting. Naively we would just load it into memory, count the total number of elements in the data set, and pick $k$ numbers from $[1...n]$.

Breaking down the problem

The first thing we need to do is consider cases where $k,n$ are small.

Lets consider $k=1$ for now and abstract it to $k=k$ later. We should also consider the cases where $n$ is still reasonable. It should work for small $n$ before it works for big $n$.

Case 1:

If $k=1, n=1$. We place the first element in the reservoir and find that we are done.

Case 2:

If $k=1, n=2$. We place the first element in the reservoir. Secondly, we know in advance that the last element must have an acceptance probability of $P=\frac{1}{2}$. When the stream ends after 2, it is clear that both elements had a $\frac{1}{2}$ probability of being in the reservoir.

Case 3:

If $k=1, n=3$. We do what we did before but we know that the third element must have an acceptance probability of $P = \frac{1}{3}$. This also means that the element in the reservoir has a $\frac{2}{3}$ chance of staying in the reservoir. Note that at $t=2$ it had a $\frac{1}{2}$ chance of being replaced. The probabilities telescope inwards when we calculate the probability of the first and second elements being in the reservoir.

$$ P = 1 \cdot \frac{1}{2} \cdot \frac{2}{3} = \frac{1}{3}$$

We can see that due to the telescoping effect, each element has a $1/3$ chance of being accepted.

Case n:

It seems that the probabilites of each element always 'telescopes' into $1/n$ as long as we accept new $i^{th}$ element with $P=1/i$.

$$ P = \frac{1}{2} \cdot \frac{2}{3} \cdot \frac{3}{4} \cdots \frac{n-2}{n-1} \cdot \frac{n-1}{n} = \frac{1}{n} $$

import random

def sample_single_element(stream):
    """Sample a random element from a stream of inputs with sample
    probability P=1/n where `n` is the size of the stream

    Arguments:
        stream (iterable) - data stream we wish to sample

    """

    for i, item in enumerate(stream, start=1):
        if random.random() < 1/i:
            result = item
    return result


# Example sampling 
stream = ['foo', 'bar', 'baz', 'spam', 'eggs']
print(sample_single_element(stream))

It turns out that to generalize this for the case where $k=k$, all we need to do is put the first $k$ items into your reservoir first and then sample with the following proceeding probability.

$$ P(accept) = \frac{i-k+1}{i} $$

Once an new value from the stream is accepted, all we need to do is remove one element from the reservoir (with uniform probability).

In this case we will see the following telescoping effect with the probabilities.

$$ P(keep) = \frac{k}{k} \cdot \frac{1}{k} \cdot \frac{k}{k+1} \cdot \frac{k+1}{k+2}\cdot \frac{k+3}{k+4} \cdots \frac{k+n-k-1}{n} = \frac{k}{n}$$

So cool!

To implement this, we will do one small trick to make the replacement a bit more readable. Instead of calculating (i-k+1/i) at each iteration, we will use randint $[1, i]$ and use that random value as a way to select which element we select.

def sample_k_elements(stream, k):
    """Sample k random elements from a stream of inputs with sample
    probability P=1/n where `n` is the size of the stream

    Arguments:
        stream (iterable) - data stream we wish to sample
        k (int) - size of reservoir

    """
    samples = list()
    for index, line in enumerate(stream):
        if index < k:
            samples.append(line)
        else:
                r = random.randint(0, index)
                if r < k:
                    samples[r] = line
    return samples


# Example sampling 
stream = ['foo', 'bar', 'baz', 'spam', 'eggs']
print(sample_k_elements(stream, k=2))

['baz', 'bar']

Conclusions

If you are interested, I recently published a python package called Reservoir that you can download on pypi. It essentially implements what I've done here but encapsulated with OOP so you can work with sampling any iterable / generator. It also comes with a cli interface if you need to sample files or data from streamed data like twitter.

You can install it with pip install Reservoir and check out the documentation on github here.
